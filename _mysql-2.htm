1   having子句

用来对多行函数结果进行过滤

 

 

l  having 和 where 作用相同，都是条件过滤

l  where 过滤普通条件，最早执行

l  having 过滤多行函数结果，分组，求完多行函数后，才执行

l  having 跟在 group by 后面

 

-- 只有一个手下的主管id

select mgr_id, count(*) c

from emps

where mgr_id is not null

group by mgr_id

having c=1;

 

-- 平均工资小于等于5000的岗位代码

select job_id, avg(sal) a

from emps

group by job_id

having a<=5000;

3   条件子查询

一个查询的查询结果，作为另一个查询的过滤条件

 

select .. where a=(select ...)

 

l  单值子查询

n  = > >= < <=

l  多值子查询

n  in

n  > all 比最大值大

n  > any 比最小值大

l  多列子查询

n  where (a,b) in (select ...)

n  where (a,b) = (select ...)

6   多表连接查询

l  两张表，按指定条件，连接成一张表

l  从连接的结果表中查询

 

 

 

use hr;

 

-- 部门表

select * from depts;

 

-- 查询员工，显示员工的部门名

select e.id,e.fname,e.sal,

        d.dept_id,d.dept_name

from emps e, depts d

where e.dept_id=d.dept_id;

 

-- 地区表

select * from locations;

 

-- 查询部门，显示部门的城市

select d.dept_id, d.dept_name,

        l.city

from depts d, locations l

where d.loc_id=l.loc_id;

 

-- 查询部门，显示部门经理名

select d.dept_id, d.dept_name,

        e.fname

from depts d, emps e

where d.mgr_id=e.id;

 

-- 查询员工，显示员工主管名

-- 自连接，一张表，看做是两张表来连接

select e1.id, e1.fname, e1.sal,

        e2.fname mgr

from emps e1, emps e2

where e1.mgr_id=e2.id;

 

 

 

 

 

7   外连接

l  内连接

n  只查询满足连接条件的数据

l  外连接

n  不满足连接条件的数据也要查询

n  左外连接

u  查询左侧表条件外数据

n  右外连接

u  查询右侧表条件外数据

n  全外连接

u  双侧表条件外数据

u  MySql不支持

 

 

 

 

l  外连接，非标准sql连接语句，每种数据库外连接语法都不同

n  sql server

where a.id(+) = b.xid

 

n  oracle

where a.id *= b.xid

 

n  MySql

没有外连接的非标准语法

 

l  标准的表连接语法

n  select ...

from a join b

     on(a.id=b.xid)

     join c

     on(...)

n  select ...

from a left join b

     on(a.id=b.xid)

n  select ...

from a right join b

     on(a.id=b.xid)

 

 

-- 所有部门，显示部门经理，没有经理显示null

 

select d.dept_id, d.dept_name,

        e.fname

from depts d

left join emps e

on(d.mgr_id=e.id);

 

-- 查询107个员工，显示部门名和城市

select e.id, e.fname, e.sal,

        d.dept_name,

        l.city

from emps e

      left join depts d

      on(e.dept_id=d.dept_id)

      left join locations l

      on(d.loc_id=l.loc_id);

8.4     事务特性ACID

l  A - Atomic

原子性，数据操作的最小单元是事务，而不是sql语句

l  C - Consistency

一致性，事务完成前后，数据要保持逻辑的一致性

A+B=100 转账前

A+B=100 转账后

l  I - Isolation

隔离性，一个事务操作数据时，其他事务要等待

l  D - Durancy

持久性，事务成功，数据被永久的保存

 

 

9   数据访问冲突的问题

三种问题

l  脏数据

l  不可重复读

l  幻觉读

 

9.1     脏数据

一个事务未提交的数据，被另一个事务查询到

9.2     不可重复读

l  针对数据修改 update 操作

l  查询过的数据，

再次查询时，与之前的查询结果不一致

9.3     幻觉读

l  针对数据的插入和删除

l  新插入的数据，查询不到

l  删除的数据，能查询到

 

10     隔离级别

为了避免数据访问冲突的问题，避免数据的混乱，数据库会让事务进行隔离

为了兼顾数据安全，和性能，数据库提供了四种隔离级别

l  级别越高，越安全，性能越差

l  级别越低，越不安全，性能越高

 

 

l  mysql设置隔离级别：

 

set tx_isolation='read-uncommitted'

脏读、不可重复读、幻觉读

 

set tx_isolation='read-committed'

不可重复读、幻觉读

 

set tx_isolation='repeatable-read' 默认级别

幻觉读

 

set tx_isolation='serialiable'
